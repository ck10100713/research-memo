## 專案概述

由 Alex Pate 維護的策展清單，收錄全球 **200+ 個公開的設計系統**（Design Systems）、UI 元件庫（UI Libraries）和模式庫（Pattern Libraries）。以字母排序的 Markdown 表格呈現，每個條目標註四個維度：Components、Voice & Tone、Designers Kit、Source Code。

> Design system 的定義：「A collection of documentation on principles and best practices, that helps guide a team to build digital products.」

## 目前限制 / 注意事項

- **純清單，無深度評測** — 只有名稱 + 連結 + 標籤，沒有比較分析
- **無分類/分組** — 200+ 條目按字母排序，無產業或框架分類
- **部分連結可能失效** — 企業併購或停維後網址變更
- **open source 不等於可自由使用** — README 特別提醒需自行確認 license
- **不含 AI-native 格式** — 傳統連結清單，非 DESIGN.md 等 AI Agent 可消費格式

## 研究價值與啟示

### 關鍵洞察

1. **23K stars 證明「索引」本身就是產品。** 這個 repo 不含任何程式碼，只有一份 Markdown 表格——但它的價值在於策展和組織。在資訊爆炸時代，「幫你找到正確資源」比「再做一個新資源」更有價值。

2. **設計系統是前端工程的「基礎設施」。** 從科技巨頭到政府機關都在投入設計系統，這不是時尚——而是前端規模化的必要條件。NASA 和愛沙尼亞政府都有自己的設計系統，說明這已超越商業需求。

3. **與 awesome-design-md 是互補關係。** awesome-design-systems（本專案）是「人類查閱設計系統的索引」，awesome-design-md（已有筆記）是「AI Agent 消費設計規範的資料庫」。前者收錄 200+ 系統的網站連結，後者收錄 58+ 品牌的 DESIGN.md 檔案。

### 與其他專案的關聯

- **vs Awesome DESIGN.md（筆記庫中）：** 完全不同的專案。本專案 2017 年起、23K stars、200+ 條目的傳統 Awesome List。awesome-design-md 是 2025 年起、39K stars、專為 AI Agent 設計的 DESIGN.md 合集。
- **對前端開發的價值：** 做 UI 設計時，先在這裡找到同產業的設計系統參考，再用 awesome-design-md 的 DESIGN.md 餵給 AI Agent 實作。
