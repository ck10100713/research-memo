## 專案概述

TradingAgents 是一個多代理 LLM 金融交易框架，模擬真實交易公司的運作方式。透過部署專門的 LLM 驅動代理（基本面分析師、情緒專家、技術分析師、交易員、風險管理團隊），平台能夠協作評估市場狀況並做出交易決策。

這個專案的創新之處在於讓多個 AI Agent 進行動態討論，以找出最佳策略。它將複雜的交易任務分解為專門的角色，確保系統具有穩健、可擴展的市場分析和決策能力。

適合場景：
- 量化交易策略研究
- 多代理系統架構參考
- LLM 在金融領域的應用研究
- AI Agent 協作模式學習

**重要提醒**：此框架僅供研究用途，不構成金融、投資或交易建議。

## 核心功能

1. **分析師團隊**：基本面、情緒、新聞、技術分析四種專門分析師
2. **研究員團隊**：多空研究員透過結構化辯論平衡風險與收益
3. **交易員代理**：整合分析報告做出交易決策
4. **風險管理**：持續評估投資組合風險和市場波動
5. **投資組合經理**：批准/拒絕交易提案的最終決策者
6. **動態討論**：代理間透過辯論找出最佳策略
7. **多 LLM 支援**：支援 o1-preview、gpt-4o、Claude 等模型
8. **CLI 介面**：互動式命令列工具進行交易分析

## 安裝與使用

### 安裝方式

```bash
# 克隆專案
git clone https://github.com/TauricResearch/TradingAgents.git
cd TradingAgents

# 建立虛擬環境
conda create -n tradingagents python=3.13
conda activate tradingagents

# 安裝依賴
pip install -r requirements.txt
```

### 環境設定

```bash
# 設定 API 金鑰
export OPENAI_API_KEY=$YOUR_OPENAI_API_KEY
export ALPHA_VANTAGE_API_KEY=$YOUR_ALPHA_VANTAGE_API_KEY

# 或使用 .env 檔案
cp .env.example .env
```

### CLI 使用

```bash
# 啟動 CLI
python -m cli.main
```

### Python 使用

```python
from tradingagents.graph.trading_graph import TradingAgentsGraph
from tradingagents.default_config import DEFAULT_CONFIG

# 初始化
ta = TradingAgentsGraph(debug=True, config=DEFAULT_CONFIG.copy())

# 執行分析
_, decision = ta.propagate("NVDA", "2024-05-10")
print(decision)
```

### 自訂配置

```python
from tradingagents.default_config import DEFAULT_CONFIG

config = DEFAULT_CONFIG.copy()
config["deep_think_llm"] = "gpt-4.1-nano"
config["quick_think_llm"] = "gpt-4.1-nano"
config["max_debate_rounds"] = 1

config["data_vendors"] = {
    "core_stock_apis": "yfinance",
    "technical_indicators": "yfinance",
    "fundamental_data": "alpha_vantage",
    "news_data": "alpha_vantage",
}

ta = TradingAgentsGraph(debug=True, config=config)
```

## 研究心得

TradingAgents 是一個設計精良的多代理金融交易框架，展示了 LLM Agent 在專業領域的應用模式。

**架構亮點：**
1. **角色專業化**：每個代理有明確的專業職責
2. **辯論機制**：多空研究員的辯論確保決策平衡
3. **分層決策**：從分析到交易再到風控的完整流程
4. **可配置性**：LLM、數據源、辯論輪數都可自訂

**對 AI Agent 開發的啟示：**
- 複雜任務可分解為專門的子代理
- 辯論/協作機制可提升決策品質
- 風險管理應該是系統設計的一部分
- 明確區分研究用途和生產用途

**使用建議：**
- 測試時使用 `gpt-4.1-mini` 等低成本模型
- 正式研究使用 `o1-preview` 等深度思考模型
- Alpha Vantage 免費版有 API 限制，生產環境考慮付費版

**限制與注意：**
- 大量 API 呼叫，成本可能較高
- 僅供研究，不構成投資建議
- 交易績效受多種因素影響
